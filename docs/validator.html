Validator

Makes simple client-side form validation easy, whilst still offering plenty of customization options. The widget comes bundled with a useful set of validation methods, including URL, email, hex color, min and max values, length validation.

How to

To activate Validator add attribute data-role="validator" to form and define validation functions over attribute data-validate="..." for inputs.

Input correct name with min length 6 symbols
Input correct email address 

                    <form data-role="validator" action="javascript:">
                        <div class="row mb-2">
                            <div class="cell-md-6">
                                <label>First name</label>
                                <input type="text" data-validate="required, minlength=6">
                                <span class="invalid_feedback">
                                    Input correct name with min length 6 symbols
                                </span>
                            </div>
                            <div class="cell-md-6">
                                <label>Email</label>
                                <input type="email" data-validate="required, email" data-role="input">
                                <span class="invalid_feedback">
                                    Input correct email address
                                </span>
                            </div>
                        </div>
                        <button class="button primary">Submit</button>
                    </form>

Functions

Metro 4 Validator supports 12 validation functions:

  • required: required value
  • minlength: min length in symbols
  • maxlength: max length in symbols
  • min: value must be more or equal then min
  • max: value must be less or equal then max
  • email: valid email required
  • url: valid url required
  • date: valid date required
  • number: value must be a number
  • digits: value must contains only digits
  • digits: value must contains color in hex
  • pattern: value must be equal with regexp pattern

To define validation for input add to input attribute data-validate="...". You can combine validation functions. To set function argument use record func_name=arg. 


                    <input type="text" data-validate="required">
                    <input type="text" data-validate="minlength=6">
                    <input type="text" data-validate="number, min=10">
                    <input type="text" data-validate="pattern=^\d+$">

Events

When Validator is works, fired any events: onBeforeSubmit(form), onError(element, value), onValid(element, value), onSubmit(form), onCreate(form).

To use event add attribute data-on-* to form.

Input correct name with min length 6 symbols 

                    <form data-role="validator" action="javascript:"
                          data-on-submit="alert('Form submitted!')">
                        <div class="row mb-2">
                            <div class="cell-md-6">
                                <label>First name</label>
                                <input type="text" data-validate="required, minlength=6">
                                <span class="invalid_feedback">
                                    Input correct name with min length 6 symbols
                                </span>
                            </div>
                        </div>
                        <button class="button primary">Submit</button>
                    </form>

Submit form

The following conditions must be met to send the form:

  1. The all the validations must be passed
  2. If an onBeforeSubmit event is defined, it must return true
